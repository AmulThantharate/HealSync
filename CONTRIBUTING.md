# Contributing to HealSync

Thank you for your interest in contributing! We welcome contributions to help improve the HealSync Disaster Recovery system.

## 🚀 How to Contribute

### 1. Reporting Bugs
- Use the GitHub issue tracker.
- Describe the bug clearly with steps to reproduce.
- Mention your environment (Docker, K8s, AWS region).

### 2. Suggesting Enhancements
- Open a new issue with "Feature Request" in the title.
- Explain why the feature is needed and how it fits the project goals.

### 3. Pull Requests
- Fork the repository.
- Create a feature branch (`git checkout -b feature/amazing-feature`).
- Ensure code follows the project's style (Flask/PEP8).
- Include tests if possible.
- Submit a Pull Request targeting the `main` branch.

## 🏗️ Development Setup

Refer to the [README.md](README.md) and [COMMANDS_ALL_IN_ONE.md](COMMANDS_ALL_IN_ONE.md) for local testing instructions.

## 🧪 Testing Guidelines
- Run Flask unit tests: `pytest app/tests`
- Verify replication logic manually using the local Docker Compose setup.
